# Quick Notes

The **Notes** tab is your notebook inside Continuity. Capture loose ideas, group them into *collapsible sections* Notion-style, write them with Markdown, and link them to a project or keep them standalone.

---

## 1. List & filters

[image: notes-01-overview.png]

All your notes appear as cards, with a search box and quick filters up top. Each card shows the title, a body snippet, its category or project, and the section count; the colored stripe on the left is its category's.

**How to use it**
- The **search** box filters by text inside titles *and* section bodies.
- The **filters** are: **All**, one per category, **Standalone** (no category), and **Pinned**.
- Tap a card to open that note in the editor.

**How to edit it**
- The green **New note** button (right of the search box) and the **+ FAB** on mobile create a note.
- If there are no notes you'll see **“No notes yet.”** with a *Create first note* button; searching with no results shows **“No notes match…”**.

> **Note:** new users get a sample note seeded for them, so it's normal to see a note already there on first open.

---

## 2. Two-panel editor

[image: notes-02-editor.png]

When you open a note, the list shrinks to a compact panel and the editor appears beside it. The title is edited up top; below it, the category and project selectors, then the sections.

**How to use it**
- Tap a list **card** to open its editor on the right.
- Type the **title** at the top.
- The editor holds: **pin** 📌, **delete** 🗑, and **close** ✕.

**How to edit it**
- Everything saves **automatically** on blur; there's no “Save” button.
- On desktop you close with the **✕**; on mobile, with the back arrow **←** (the editor goes full-screen).

---

## 3. Collapsible sections

[image: notes-03-sections.png]

Each note's body is organized into toggle-style blocks. In **view** mode the Markdown is rendered (bold, lists, code). Each section carries: eye 👁, ▲▼ arrows, grip ⠿, and trash 🗑.

[image: notes-04-sections.png]

Tap the pencil ✎ (or the text itself) to edit in a monospace *textarea*; the eye 👁 switches back to the rendered view.

**How to use it**
- The **chevron** ▸/▾ on the left **collapses and expands** the section.
- Type the section **title** in the top bar.
- The body supports **Markdown**: bold, italic, `code`, lists, headings, and links.
- **Add collapsible section** (the dashed button at the bottom) adds a new block.

**How to edit it**
- The **eye** 👁 / **pencil** ✎ toggles between **view** (Markdown) and **edit** (textarea).
- Reorder with the **grip** ⠿ (drag & drop) or the **▲▼** arrows as a keyboard fallback.
- The **trash** 🗑 deletes that section; everything saves on blur.

---

## 4. Category & project

[image: notes-05-categories.png]

Classify each note with a color category and, optionally, link it to a project. They're two dropdowns in the editor header.

**How to use it**
- Open **Category** and pick one; the card's stripe and chip take that color.
- Open **Project** to hang the note off a project and find it in context.
- Filter the list by a category or by **Standalone** to see only the uncategorized ones.

**How to edit it**
- Pick **No category** / **No project** to unlink.
- Categories are created and edited like elsewhere in the app (shared with tasks and projects).

---

## 5. Pinning notes

[image: notes-06-pinned.png]

Flag your most important notes to keep them one tap away with the **Pinned** filter. A filled 📌 icon shows on the card.

**How to use it**
- Open the note and tap the **📌 Pin note** icon in the editor header.
- Pinned notes show a **filled pin** on their list card.
- Use the **Pinned** filter to see only what you've anchored.

**How to edit it**
- Tap the pin again to **unpin**.
- Pinning doesn't change the category or project: it's a separate marker.

---

## 6. Creating a note

[image: notes-07-create.png]

There are three ways to start a new note: the **New note** button (desktop), the empty-state *Create first note* button, and the **+ FAB** (mobile).

**How to use it**
- **New note** instantly opens an empty note with its first section ready.
- If you create from a **category filter**, the note is born with that category.
- On mobile, the **+ FAB** expands this action alongside access to **Loop**.

**How to edit it**
- A note with no title is listed as **“Untitled note”** until you give it one.
- Just start writing: title, sections, and Markdown all save themselves.
