# Projects

The **Projects** tab is your master list: every project in one table, with its status, priority, category, task progress and next step. From here you search, filter, sort, open each project to see its detail, and create new ones.

---

## 1. The project list

[image: projects-01-overview.png]

Top to bottom, the view is built like this:

1. **Header** — the “All Projects” title, the search box and the **New** button (on desktop).
2. **Filter bar** — status, category, priority and sort (section 2). On mobile these become two buttons that open sheets.
3. **The list of rows** — one per project: status icon, name, chips and task progress.
4. **Create button (FAB)** — floating bottom-right to add a project without leaving the view.

> **Note:** the default order is **Smart** — projects with overdue tasks first, then due-today, then ones that have gone idle, then the rest, within each group by priority and recent activity.

---

## 2. Filter & search bar

[image: projects-02-filters.png]

Narrow the list by status, category, priority and due date, or search by text. On desktop each chip shows **how many projects** fall under it.

**How to use it**
- The search box filters by name, description, next step, motivation and category.
- Tap a **Status**, **Category** or **Priority** chip to filter; go back to “All” to clear it.
- Statuses with no projects aren't shown; the number on each chip is its count.
- The **Sort** selector changes how the list is arranged (section 4).

**How to edit it**
- Filters are **cumulative**: status + category + priority + due date at once.
- If nothing matches, a **“Clear filters”** button appears to reset to “All”.

---

## 3. Filter sheet (mobile)

[image: projects-03-filtersheet.png]

On mobile, “Filter” opens a sheet with every option. The bottom button shows **how many projects would match** before you apply; if it's 0 it reads “No projects match”.

**How to use it**
- Open it with the **Filter** button; its badge shows how many filters are active.
- Pick one per group: **Status**, **Priority**, **Category** (if you have any) and **Due date**.
- The **Apply (N)** button carries the live count; **“Clear”** resets back to All.

**How to edit it**
- **Due date**: Any, Overdue, Next 7 days or No date.
- On desktop these same filters live in the labeled bar (section 2), with no sheet.

---

## 4. Sorting projects

[image: projects-04-sort.png]

Five modes to arrange the list. The active option carries a checkmark; on desktop it's the “Sort” dropdown.

**What each mode does**
- **Smart** — overdue first, then due today, then idle, then by priority.
- **By priority** — Critical → High → Medium → Low.
- **Recent activity** — most recently touched on top.
- **Name** — alphabetical.
- **By status** — groups by status in its natural order.

**How to edit it**
- The chosen mode persists while you stay in the view.

## 5. Anatomy of a row

[image: projects-05-card.png]

Everything a project tells you at a glance in the list: left to right, priority stripe, chevron, status icon, name + chips, and the task progress bar.

**What it shows**
- **Color stripe** on the left = priority (red critical, orange high, green medium, blue low).
- **Square icon** = project status (Active, Stalled, Idea, Launched…).
- **Category chip** in its color, if the project has one.
- **Urgency badge**: **N overdue**, **N today**, **Nd idle** or **Nh pending**.
- **→ Next step** under the name, and the **done/total** task bar on the right.

**How to use it**
- **Tap the row** to expand the project detail (section 6).
- The amber “idle” badge is just a visual hint (7+ days), not a status.
- On mobile, category and idle hide to save space; progress collapses to “done/total”.

---

## 6. Opening a project

[image: projects-06-detail.png]

Tapping a row expands it **inline** (the row lights up), with next step, tasks and activity. “Open project” takes you to the full-screen view.

**How to use it**
- **Next step** on top, always visible; below it Why this matters, Description, Tasks, Activity and Notes.
- In **Tasks**: check the ✓ to complete, **Add task**, or edit/delete each one.
- **Log update** leaves a log entry and “wakes” the project up.
- **Open project** (↗) jumps to the full detail view.

**How to edit it**
- **Edit** opens the project form (name, status, priority, category, dates…).
- **Delete** removes the project (asks for confirmation).
- Long task or activity lists are trimmed with a “show more”.

---

## 7. Creating & empty state

[image: projects-07-create.png]

Before you have any projects, you see the empty state with the **“Add your first project”** shortcut.

[image: projects-08-create.png]

On mobile, the floating **+** button reveals “New project” and access to **Loop** (the AI assistant).

**How to use it**
- On desktop, the header's **New** button opens the project form.
- On mobile, the floating **+** button does the same (and offers to open Loop).
- The empty state includes a direct shortcut: **“Add your first project”**.

**How to edit it**
- If filters leave the list empty you'll see **“No projects match these filters”** with **Clear filters**.
- A search with no results shows the searched text so you can adjust the term.
